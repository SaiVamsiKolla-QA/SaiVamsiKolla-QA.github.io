About Me

I'm a Quality Engineer at mimik, owning the test strategy for an on-device AI health app end to end — validating AI agent behavior, model responses, and the ingestion of health data from Apple HealthKit.

I enjoy finding bugs before customers do, but even more, I enjoy building tools that help teams find them faster. Whether it's creating automation frameworks, writing API tests, or developing utilities that make testing easier, I like solving quality problems through engineering.

Projects

Testing tools I built to solve problems I kept running into.

qa-agent

Testing an LLM is hard because the same question gives a different answer each run, so a normal assertion can't tell a regression from noise. qa-agent retrieves ISTQB context locally and scores each answer against a golden set with DeepEval, then fails the CI build when quality drops below a set threshold. It's regression testing for non-deterministic output, running offline on a small model to stay reproducible. 59 tests.

  • Python
  • ChromaDB
  • DeepEval
  • RAG
View on GitHub →

fhir-contract-fuzz

Healthcare APIs have to match the FHIR R4 spec exactly, and a subtle contract drift breaks downstream systems quietly. The framework pins the contract with pydantic models and fuzzes the live OpenAPI surface with Schemathesis, split into a fast PR gate and a nightly deep run. The architecture and CI are in place; I'm growing coverage against a live HAPI FHIR server.

  • PyTest
  • Schemathesis
  • pydantic v2
  • Docker
View on GitHub →

medusa-checkout-e2e

Checkout is where e-commerce quality actually matters and the flow most likely to break on a release. The suite runs against a real Dockerized Medusa store instead of mocks, with page objects isolating selectors and Playwright auto-waiting to kill flaky sleeps. Built as a framework foundation with CI and Allure reporting wired end to end.
